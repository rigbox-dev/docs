# Rigbox

Rigbox is an open platform for running untrusted code in secure, isolated [Firecracker](https://firecracker-microvm.github.io/) micro-VMs. Each workspace is a full Linux VM with SSH access, app routing, snapshots, and live metrics — provisioned and managed entirely through the API.

  New to Rigbox? Start with the [Quickstart](/quickstart) to deploy your first workspace in under a minute.
</Tip>

## Architecture

Rigbox uses a two-zone security model:

- **Control Plane** — holds agent runtimes, API keys, and user state. Runs the API gateway, auth service, and admin portal.
- **Data Plane** — runs untrusted generated code in Firecracker VMs with no access to platform secrets.

All communication between zones goes through authenticated internal APIs over a Tailscale mesh.

## Base URL

```bash
https://api.rigbox.dev/api
```

All endpoints are relative to this base. Authentication is required for most endpoints — see [Authentication](/authentication) for details.

## Response format

All responses are JSON. Errors include a `message` field:

```json
{
  "message": "Workspace not found"
}
```

| Status | Meaning |
| --- | --- |
| `2xx` | Success |
| `401` | Missing or invalid authentication |
| `403` | Insufficient permissions |
| `404` | Resource not found |
| `429` | Rate limited — retry with exponential backoff |
| `5xx` | Server error |
