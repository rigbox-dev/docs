# SSH Access

Rigbox provides SSH access to your workspaces through a dedicated SSH gateway on port 2200. You can open a terminal session, transfer files, and even access an interactive TUI dashboard — all over SSH.

<Warning>
  Rigbox uses port **2200** for SSH, not the standard port 22. Make sure to specify `-p 2200` in all SSH commands.
</Warning>

## Prerequisites

Before connecting via SSH, you need:

1. An SSH key pair on your local machine (e.g., `~/.ssh/id_ed25519`)
2. A Rigbox account with an API key
3. At least one running workspace

If you don't have an SSH key yet, generate one:

```bash
ssh-keygen -t ed25519 -C "your-email@example.com"
```

---

## Register an SSH Key

To connect via SSH, you first need to register your public key with Rigbox.

### Via the API

Send your public key to the SSH keys endpoint:

<CodeGroup>

```bash cURL
curl -X POST https://api.rigbox.dev/api/ssh-keys \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "laptop",
    "public_key": "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA... your-email@example.com"
  }'
```

```javascript JavaScript
const response = await fetch("https://api.rigbox.dev/api/ssh-keys", {
  method: "POST",
  headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    name: "laptop",
    public_key: "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA... your-email@example.com",
  }),
});

const key = await response.json();
console.log("Registered key:", key.id);
```

```python Python
import requests

response = requests.post(
    "https://api.rigbox.dev/api/ssh-keys",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    json={
        "name": "laptop",
        "public_key": "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA... your-email@example.com",
    },
)

key = response.json()
print(f"Registered key: {key['id']}")
```

</CodeGroup>

### Via the CLI

If you have the Rigbox CLI installed:

```bash
rig ssh-key add laptop
```

This reads your default public key (`~/.ssh/id_ed25519.pub`) and registers it automatically.

<Tip>
  You can register multiple SSH keys for different machines — for example, one for your laptop and one for your desktop. Give each a descriptive name so you can identify them later.
</Tip>

## Sync Keys to a Workspace

When you create a workspace, your registered SSH keys are typically synced automatically. However, if you add a new key while a workspace is already running, you need to sync it manually.

<CodeGroup>

```bash cURL
curl -X POST https://api.rigbox.dev/api/workspaces/{workspace_id}/ssh-keys/sync \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
await fetch(
  `https://api.rigbox.dev/api/workspaces/${workspaceId}/ssh-keys/sync`,
  {
    method: "POST",
    headers: { "Authorization": "Bearer YOUR_API_KEY" },
  }
);
```

```python Python
requests.post(
    f"https://api.rigbox.dev/api/workspaces/{workspace_id}/ssh-keys/sync",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
)
```

</CodeGroup>

This copies all your registered public keys into the workspace VM's `~/.ssh/authorized_keys` file.

<Note>
  Keys are synced automatically on workspace start for most templates. You only need the sync endpoint if you add a key to an already-running workspace.
</Note>

## Connect to a Workspace

Once your key is registered and synced, connect as the `developer` user:

```bash
ssh developer@rigbox.dev -p 2200
```

The SSH gateway identifies you by your key and routes the connection to your workspace. If you have multiple workspaces, the gateway connects you to the most recently active one.

### Specify an identity file

If your key is not in the default location, specify it explicitly:

```bash
ssh -i ~/.ssh/my_rigbox_key developer@rigbox.dev -p 2200
```

### SSH config shortcut

Add this to your `~/.ssh/config` for easier access:

```ssh-config
Host rigbox
  HostName rigbox.dev
  User developer
  Port 2200
  IdentityFile ~/.ssh/id_ed25519
```

Then connect with:

```bash
ssh rigbox
```

## TUI Dashboard

Rigbox includes an interactive terminal dashboard accessible over SSH. Connect as the `rig` user instead of `developer`:

```bash
ssh rig@rigbox.dev -p 2200
```

The TUI dashboard shows:

- **Workspace list** — all your workspaces with status indicators
- **Resource usage** — CPU, memory, and disk for each workspace
- **App status** — running apps, their ports, and health
- **Quick actions** — start, stop, and manage workspaces

Navigate with arrow keys, press Enter to select, and `q` to quit.

<Tip>
  The TUI dashboard is a great way to check on your workspaces without leaving the terminal. It updates in real time and works over any SSH connection, including from mobile terminal apps.
</Tip>

## File Transfer

Use `scp` to transfer files between your local machine and a workspace. Remember to specify port 2200 with the `-P` flag (uppercase for `scp`).

### Upload a file

```bash
scp -P 2200 ./local-file.txt developer@rigbox.dev:/home/developer/
```

### Download a file

```bash
scp -P 2200 developer@rigbox.dev:/home/developer/output.txt ./
```

### Upload a directory

```bash
scp -P 2200 -r ./my-project developer@rigbox.dev:/home/developer/
```

### Using rsync

For larger transfers or syncing directories, `rsync` is more efficient:

```bash
rsync -avz -e "ssh -p 2200" ./my-project/ developer@rigbox.dev:/home/developer/my-project/
```

<Warning>
  The SSH gateway routes based on your key. If you have multiple workspaces, make sure you are connected to the correct one before transferring files.
</Warning>

## Managing SSH Keys

### List your registered keys

<CodeGroup>

```bash cURL
curl https://api.rigbox.dev/api/ssh-keys \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```bash CLI
rig ssh-key ls
```

</CodeGroup>

Response:

```json
[
  {
    "id": "key_abc123",
    "name": "laptop",
    "fingerprint": "SHA256:...",
    "created_at": "2026-03-15T10:30:00Z"
  },
  {
    "id": "key_def456",
    "name": "desktop",
    "fingerprint": "SHA256:...",
    "created_at": "2026-04-01T14:20:00Z"
  }
]
```

### List keys synced to a workspace

```bash
curl https://api.rigbox.dev/api/workspaces/{workspace_id}/ssh-keys \
  -H "Authorization: Bearer YOUR_API_KEY"
```

This returns the keys currently in the workspace's `authorized_keys` file, which may differ from your registered keys if you haven't synced recently.

### Remove a key

<CodeGroup>

```bash cURL
curl -X DELETE https://api.rigbox.dev/api/ssh-keys/{key_id} \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```bash CLI
rig ssh-key rm laptop
```

</CodeGroup>

<Warning>
  Removing a key from the API does not immediately remove it from running workspaces. The key will be removed the next time the workspace restarts or you manually trigger a sync. To revoke access immediately, sync keys after deletion.
</Warning>

## Troubleshooting

### Connection refused

If you get `Connection refused`, verify:

1. You are using port 2200: `ssh -p 2200 developer@rigbox.dev`
2. Your workspace is running (check with `GET /api/workspaces/{id}` or `rig ls`)
3. Your SSH key is registered and synced

### Permission denied

If you get `Permission denied (publickey)`:

1. Check that your key is registered: `GET /api/ssh-keys`
2. Sync keys to the workspace: `POST /api/workspaces/{id}/ssh-keys/sync`
3. Verify you are using the correct identity file: `ssh -v -p 2200 developer@rigbox.dev`

### Slow connection

The SSH gateway adds minimal latency. If connections feel slow:

1. Check your network connection
2. Try connecting with `-o ServerAliveInterval=60` to prevent timeouts
3. For file transfers, prefer `rsync` over `scp` for large datasets
